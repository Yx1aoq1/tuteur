# Import 风格规则

本文件定义 `code-style` 的 `default TS/JS preset` 中的 `import` 规则。

## 适用范围

- 目标范围内的 `*.ts`、`*.tsx`、`*.js`、`*.jsx`、`*.mts`、`*.cts` 源文件
- 其他存在 ES Module `import` 的 TS/JS 文件

## 执行原则

- 仅做最小改动，不改业务语义。
- 规则必须可机械执行、可复现，避免主观排序。
- 执行后检查 diff 范围，防止非目标文件污染。

## 期望输出

- 仅改 `import` 相关顺序、分组、空行与格式。
- 不新增/删除导入成员（除非用户明确要求）。
- 保持最小 diff。

## 规则

1. 按命名风格分组

- 分组顺序：`PascalCase` -> `camelCase` -> `UPPER_SNAKE_CASE`。
- 组间保留 1 个空行。

2. 组内排序

- 按整行字符长度从长到短排序。
- 若长度相同，按导入成员名字母序（A-Z）排序。

3. `type` 导入

- 同一命名组内，`import type` 在值导入前。

4. 单行成员规则

- 单行 `import { ... } from '...'` 内成员按成员名长度从长到短排序。
- 成员名长度相同按字母序（A-Z）。

5. 约束

- 不改路径别名与路径文本。
- 不改默认导入、命名导入、类型导入的语义。
- 不引入 lint 无关重构。
