# 检查清单

以下清单用于 `code-style` 的 `default TS/JS preset` 收尾核查。

在完成规则应用后，执行以下检查：

1. 变更范围

- 仅修改用户指定文件或目录。
- 未引入业务逻辑改动。

2. 规则层确认

- 明确本次命中的规则层（`import`、`export/barrel`、`comment-doc`、`layout-readability`）。
- 最终输出需说明命中的规则层与自检结果。
- 只检查本次命中的规则层，不把纯排版请求升级成文档补充任务。

3. `import` 规则（命中时）

- `import` 的分组、排序、空行符合对应引用文档。
- 对于 `import`，命名分组顺序为 `PascalCase` -> `camelCase` -> `UPPER_SNAKE_CASE`。
- 组间保留 1 个空行。

4. `export/barrel` 规则（命中时）

- `export` 的分组、排序、空行符合对应引用文档。
- 单行 / 多行 `export` 的分层和成员排序符合规则。

5. `comment-doc` 规则（命中时）

- 本次范围内已纳入 `comment-doc` 覆盖范围、且需要补充 JSDoc 的函数，已完成最小必需注释。
- 本次范围内已纳入 `comment-doc` 覆盖范围且涉及参数的函数包含 `@param`。
- 本次范围内需要额外说明返回语义的函数，已使用 `@return` 补充说明，且未使用 `@returns`。
- TypeScript 文件中的 `@return` 未机械重复签名里的类型声明，优先说明返回语义、约束、单位或边界条件。
- 返回语义显然或只会重复类型签名的函数，没有机械补 `@return`。
- 本次范围内已纳入 `comment-doc` 覆盖范围、且命中“对外 API / 复用 Hook / 复用工具函数”条件的函数，已按规则补齐 `@example`。
- `interface/type/enum/const` 顶层说明采用正确的注释形态。
- 单行字段注释统一使用 `//`；仅多行字段说明使用 `/** ... */`。
- `enum` 顶层说明存在；成员注释在需要时保留且风格一致。
- `enum` 成员已按“注释 + 成员”为组排版，组间保留 1 个空行；单行注释使用 `//`，多行说明使用 `/** ... */`。
- `interface/type` 与内联对象字段已按“注释 + 字段”为组排版，组间保留 1 个空行；单行注释使用 `//`，多行说明使用 `/** ... */`。
- 字段注释避免无信息增量的机械注释。
- 未纳入 `comment-doc` 覆盖范围的局部函数，没有因宽泛风格请求被机械补成完整 JSDoc。
- 若未要求语义重写，已按“保守格式化”统一注释形态与结构；除规则要求的最小补充外，未无谓扩大文案改写范围或重写已有说明文案。

6. `layout-readability` 规则（命中时）

- 声明段结束后到控制流 / 副作用调用之间的空行符合对应引用文档。
- 顶层摘要注释 + 顶层 `const` 声明组之间的空行符合规则。
- `if/else if/else`、`try/catch/finally`、`switch` 等分支块结束后的空行符合规则。
- 状态对象、配置对象、普通对象字面量、对象方法成员之间的分组空行符合规则。
- 直接透传 Promise 的函数没有冗余 `async`；`async` 函数中没有无意义的 `return await`。
- 字典对象仅在用户明确要求排序时才执行排序；若未明确要求排序，保持原有顺序。
- 未命中 `comment-doc` 时，没有顺带新增 JSDoc、`@example`、顶层说明或字段说明。

7. 格式化

- 使用 `prettier --write <explicit_file_list>`，仅格式化目标文件，不做全仓格式化。
- 避免调用会隐式执行全仓格式化的包装脚本。
- `prettier`、lint、`tsc` 只能辅助验证，不能替代规则层检查。

8. 差异检查

- 使用 `git --no-pager diff -- <target_files>` 确认 diff 受控。
- 执行格式化或项目脚本前后，对比 `git status --short` baseline，识别本次命令新增的 spillover。
- 仅在可以确认不影响执行前已有改动时，才处理非目标路径污染；若无法安全分离，停止并汇报。

9. 环境检查

- 依赖可用（`node_modules` 存在或可安装）。
- `.gitignore` 存在。

10. 工具回退

- 搜索命令优先 `rg`；若不可用，使用 `find` 作为回退（fallback）。
